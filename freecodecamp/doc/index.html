Overview

The TaskFlow API is a RESTful web service that allows developers to programmatically interact with the TaskFlow project management platform. It enables the creation, retrieval, updating, and deletion of tasks, projects, and team members. The API is designed to be predictable, resource-oriented, and easy to integrate with external tools.

Base URL & Environments

All API requests must be made over HTTPS. Plain HTTP requests will be rejected. We provide two environments for integration and testing.

  • Production: https://api.taskflow-example.com/v1
  • Sandbox (Testing): https://sandbox.taskflow-example.com/v1

Authentication

The TaskFlow API uses Bearer Token authentication. You must include your secret API key in the Authorization header of every request.

Security Note: Never expose your API keys in publicly accessible areas such as GitHub, client-side code, or public forums.

# Example Authentication Header
curl -X GET "https://api.taskflow-example.com/v1/projects" \
  -H "Authorization: Bearer tf_live_xYzA123456789" \
  -H "Content-Type: application/json"

Rate Limiting

To ensure system stability, requests are capped based on your account tier. Rate limit information is returned in the HTTP response headers of every request.

  • Standard Tier: 100 requests / minute
  • Enterprise Tier: 1000 requests / minute

Rate Limit Headers

  • X-RateLimit-Limit: The maximum number of requests allowed in the current time window.
  • X-RateLimit-Remaining: The number of requests remaining in the current window.

Standard Error Handling

TaskFlow uses conventional HTTP response codes to indicate the success or failure of an API request.

  • 2xx codes Indicate success.
  • 4xx codes Indicate an error that failed given the information provided (e.g., a required parameter was omitted).
  • 5xx codes Indicate an error with our servers.

When an error occurs, the API returns a standard JSON payload detailing the issue:

{
  "error": {
    "code": "invalid_parameter",
    "message": "The 'due_date' field must be a valid ISO 8601 timestamp.",
    "status": 400
  }
}

Data Models

Before interacting with the endpoints, it is important to understand the core Task object.

{
  "id": "tsk_8f7e6d5c",
  "project_id": "prj_1a2b3c4d",
  "title": "Migrate Database to PostgreSQL",
  "status": "in_progress",
  "assignee_id": "usr_998877",
  "created_at": "2026-03-15T10:00:00Z"
}

Pagination

Endpoints that return lists of objects, such as GET /tasks, are paginated to improve performance. We use cursor-based pagination. You can control the number of items returned using the limit parameter (default: 50, max: 100).

To fetch the next page, pass the next_cursor value from the previous response into the cursor query parameter.

{
  "data": [
    /* array of task objects */
  ],
  "meta": {
    "has_more": true,
    "next_cursor": "cXdlcnR5dWlvcA=="
  }
}

Endpoint: Retrieve Projects (GET)

Fetches a list of all active projects your API key has access to.

HTTP Request

GET /v1/projects

Code Example (Python / Requests):

import requests

url = "https://api.taskflow-example.com/v1/projects"
headers = {
    "Authorization": "Bearer tf_live_xYzA123456789",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers)

if response.status_code == 200:
    print(response.json())
else:
    print(f"Failed to fetch projects: {response.status_code}")

Endpoint: Create a Task (POST)

Creates a new task within a specific project.

HTTP Request

POST /v1/tasks

Request Body Parameters:

  • project_id (string, required): The ID of the parent project.
  • title (string, required): The name of the task.
  • status (string, optional): Default is todo.

Code Example (Node.js / Fetch):

const response = await fetch('https://api.taskflow-example.com/v1/tasks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer tf_live_xYzA123456789',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    project_id: 'prj_1a2b3c4d',
    title: 'Update server dependencies',
    status: 'todo'
  })
});

const data = await response.json();
console.log(data);

Webhooks

TaskFlow can send HTTP POST payloads to your server's webhook URL whenever certain events occur (e.g., when a task is marked as "done").

You must configure your webhook endpoints in the Developer Dashboard and verify the webhook signature to ensure the payload originated from TaskFlow.

Example Webhook Payload:

{
  "event_id": "evt_55667788",
  "event_type": "task.completed",
  "created_at": "2026-03-20T11:53:00Z",
  "data": {
    "task_id": "tsk_8f7e6d5c",
    "project_id": "prj_1a2b3c4d",
    "completed_by": "usr_998877"
  }
}
